# Roblox Account Creation Automation

An automated form-filling utility using Playwright to launch clean instances of Google Chrome on macOS, dynamically generate randomized account credentials, fill out the Roblox account creation form, and log the details locally.

> [!IMPORTANT]
> **Strict Environment Support:** This script relies explicitly on the local macOS filesystem layout and standard absolute application bundle paths (`/Applications/Google Chrome.app`). It is supported **ONLY** on macOS.

## How To Make It Work?

**Step 1:** Installing Node.js on Your Mac  
Begin by installing the Node.js in **https://nodejs.org/en/download/current** runtime environment required to execute the project. Visit **nodejs.org**, download the recommended **LTS (Long Term Support)** installer for macOS, and complete the standard installation process. After installation finishes, open the **Terminal** application and run `node -v` to verify that Node.js has been installed successfully. If a version number is displayed, your system is ready for the next step. **(Please download and install Google Chrome. Note that alternative browsers may not fully support all features of this platform.)**

**Step 2:** Preparing the Project Folder  
Locate the downloaded project archive inside your **Downloads** folder and extract its contents. Rename the extracted folder to **RobloxAccountCreation** if it is not already named that, then move it to your preferred location on your Mac where you want to keep the project files.

**Step 3:** Opening the Project in Terminal  
Launch the **Terminal** application and navigate into the project directory. An easy way to do this is by typing `cd` followed by a space, then dragging and dropping the **RobloxAccountCreation** folder directly into the Terminal window to automatically insert the full file path. Press **Enter** to switch the Terminal into the project workspace.

**Step 4:** Installing Project Dependencies  
With Terminal now inside the project directory, run the command `npm install`. This allows the Node Package Manager to read the project's `package.json` file, download all required dependencies, install the Playwright automation framework, and generate the local `node_modules` directory along with the `package-lock.json` file.

**Step 5:** Starting the Automation  
Run the project using the command `npm start`. The application will launch a new isolated Google Chrome session through Playwright and navigate to the Roblox registration page. The project then executes the `fillForm.js` script, generates unique profile information, records the generated credentials inside `accounts.txt`, and automatically fills the registration form.

**Step 6:** Completing the CAPTCHA and Stopping the Program  
Since Roblox uses **Arkose Labs FunCaptcha** as part of its registration process, you'll need to complete the CAPTCHA manually in the Chrome window during the built-in **180-second** delay. After the timer expires, the browser session closes, the temporary browser profile is removed, and a new session begins automatically. If you need to stop the automation at any time, return to the **Terminal** window and press **Control + C** to terminate the running process.



## Features

- **Google Chrome Integration**: Automatically hooks into your local macOS Google Chrome application.
- **Session Isolation**: Spins up a completely isolated, fresh user profile environment on each run, mimicking a clean browser session every single time.
- **Dynamic Variable Generation**: Pure JavaScript algorithmic generation of random lowercase alphanumeric usernames (6-13 characters) appended with exactly 3 digits (e.g., `sahdkadhwqkh831`), alongside randomized secure passwords.
- **Automatic Logging**: Captures credentials inside the running context and streams them directly into a local `accounts.txt` file.
- **Automated Lifecycle**: Visibly loads the signup portal, populates inputs via a React framework value bypass, runs staggered submission hooks, waits exactly 3 minutes (you can change that in index.js) to permit manual CAPTCHA resolution, and tears down the browser context cleanly before recycling the loop.
